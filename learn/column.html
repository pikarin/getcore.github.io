Column Float, Table Cell and Flexbox

Class grid

The Core column has 3 properties, namely float, table cell and flexbox. In terms of writing Code not much different Float add class (row) Cell table (row-cell) Flexbox (row-flex) to create a column, Float add (column) Flexbox cell (cell) table (flex). To create an equally flat column, use the class grid, if using interior or Inside, it will make the distance too far. The class grid clears it up. Here's a sample of writing.

Some grid spacing you can use

  • grid-s :padding: 0 4px;
  • grid-m :padding: 0 16px;
  • grid-l :padding: 0 24px;

Flexbox (flex)

Column Flex is the 3rd standard for column Core creation. The advantage of flexbox is not to use offset to create space without content. The flexbox properties are different from floats that must use margins to push or empty content. With the flexbox of Core, we do not have to use offset, use the code as usual and see the hash. Here's the code and see the results.

<div class="row-flex"> <div class="flex l6 m6 s6"> </div> <div class="flex l6 m6 s6"> Column Cell 6 (right) </div> </div>
Column Flex 6 (right)

Wow! Very cool, but the flexbox of the cores has a problem on responsive sized smarphone. On s1 size (smartphone size 1). Exceeded the browser width limit. To overcome them use S2 at least when want to responsive look good on smartphone. Here's what not to use For responsive smartphones.

<div class="row-flex"> <div class="flex l6 m6 s1"> Column s1 on the smartphone will exceed the width of the browser (although only slightly). </div> <div class="flex l6 m6 s11"> Column Cell 6 (right) </div> </div>
<div class="row-flex"> <div class="flex l6 m6 s2"> Column s2 is the most recommended for smartphone size. </div> <div class="flex l6 m6 s10"> Column Cell 6 (right) </div> </div>
